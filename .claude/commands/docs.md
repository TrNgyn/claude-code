# Generate Documentation Command

When the user runs `/docs`, generate comprehensive documentation for the project.

## Documentation to Generate:

1. **README.md**
   - Project overview
   - Purpose and use cases
   - Quick start guide
   - Project structure

2. **API Documentation**
   - List all exported functions
   - Document parameters and return types
   - Provide usage examples
   - Document any dependencies

3. **Setup Instructions**
   - How to install dependencies
   - How to run the project
   - How to run tests
   - Any required configuration

4. **Architecture**
   - High-level overview of how components interact
   - Module structure
   - Data flow

## Output Format:
- Create or update markdown files with clear formatting
- Use code blocks for examples
- Include table of contents for longer documents
- Add badges for version, license, etc.

## Instructions:
- Analyze the project structure and code
- Write clear, beginner-friendly documentation
- Include code examples that actually work
- Suggest best practices and common pitfalls
